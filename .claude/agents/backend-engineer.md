## Project context

The backend lives at `apps/backend/`. All `dotnet` commands must be run from that directory.

Connection string key: `ConnectionStrings:Default` (configured in `appsettings.Development.json`, gitignored).

Migrations: `dotnet dotnet-ef migrations add <Name>` then `dotnet dotnet-ef database update`.

## Vertical Slice conventions

Each use case is a single file under `Features/<Domain>/`. Co-locate the request, response, handler, validator, and endpoint registration in that file. No shared service layers.

### Canonical slice structure

```csharp
namespace Backend.Features.<Domain>;

public static class <UseCaseName>
{
    public record Request(...) : IRequest<Response>;
    public record Response(...);

    public sealed class Handler(AppDbContext db) : IRequestHandler<Request, Response>
    {
        public async Task<Response> Handle(Request req, CancellationToken ct)
        {
            // ...
            return new Response(...);
        }
    }

    public static void MapEndpoint(IEndpointRouteBuilder app)
    {
        app.MapPost("/route", async (Request req, IMediator mediator) =>
        {
            var result = await mediator.Send(req);
            return Results.Ok(result);
        });
    }
}
```

Register endpoints in `Program.cs` by calling each `MapEndpoint` extension.

## EF Core conventions

- Entity configurations implement `IEntityTypeConfiguration<T>` — placed in `Infrastructure/Database/Configurations/` or co-located with the entity.
- `AppDbContext.OnModelCreating` uses `ApplyConfigurationsFromAssembly`, so new configurations are picked up automatically.
- Always use `HasKey`, explicit column types (`varchar`, `text`, `uuid`, `timestamptz`), and `IsRequired` in configurations — never rely on conventions alone.
- Prefer `Guid` primary keys generated by the database (`HasDefaultValueSql("gen_random_uuid()")`).
- Use `ct` (CancellationToken) on every async EF call.

## Code rules

- Use C# 12+ features: primary constructors, collection expressions, `required` properties.
- Records for DTOs (Request/Response). Classes for entities.
- No `async void`. No `.Result` or `.Wait()`.
- Return `IResult` from endpoints (`Results.Ok`, `Results.Created`, `Results.NotFound`, etc.).
- Validate inputs inside the handler or with a dedicated `Validator` class in the same file.
- Do not add comments that explain what the code does — only add a comment when the WHY is non-obvious.
